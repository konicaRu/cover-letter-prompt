# ARCHITECTURE — устройство проекта

> Структура и все изменения. Обновляется по команде `git save`.
> Правила работы — в CLAUDE.md, его не дублируем, тут только карта и changelog.

## Назначение
Генератор сопроводительных писем под вакансии (аналитик данных Д. Кочетов)
с итеративной доработкой промпта по замечаниям пользователя.

## Слои промпта (приоритет: core > corrections > rules)
- `prompt/core.md` — заблокированное ядро. Меняется только командой «изменить core».
- `prompt/rules.md` — правила стиля и промптинга.
- `prompt/corrections.md` — правки пользователя (append-only, применяются только `active`).
- `prompt/humanizer.md` — флаг очеловечивания (одна строка: `on`/`off`, по умолч. `off`).

## Вход / выход
- `inputs/resume/` — резюме (`.md`), источник фактов №1. PDF не читаем (медленно/дорого).
- `inputs/jira/` — задачи Jira, источник опыта №2.
- `inputs/vacancy.md` — текущая вакансия.
- `outputs/letters.md` — 3 варианта письма (плоская версия).
- `outputs/letters_humanized.md` — те же письма после скилла `humanizer` (когда флаг `on`).
- `bench/` — эталонные ПАРЫ вакансия+письмо для регресс-проверки. Одна папка на вакансию
  по короткому имени работодателя (`bench/gri/` = vacancy.md + letters.md).

## Служебные файлы
- `MEMORY.md` — журнал диалога: на чём закончили.
- `ARCHITECTURE.md` — этот файл.

## Команды
- «Сгенерируй письмо», «Замечание: …», «Проверь на противоречия»,
  «Повысить C-NNN до core», «humanizer on/off» — описаны в CLAUDE.md.
- `git save` — флаш + бэкап: (1) обновить MEMORY.md и ARCHITECTURE.md актуальным
  состоянием; (2) `git add -A` (inputs/jira исключён через .gitignore), commit, `git push origin main`.
  Репо: github.com/konicaRu/cover-letter-prompt (main, ПУБЛИЧНЫЙ). Перед коммитом —
  sanity-check, что Jira не в staged.
- `git load` — утреннее восстановление: прочитать MEMORY.md + active-правки + humanizer
  и кратко доложить, где остановились. Только чтение, ничего не пишет.

## Бэкап / git
- Удалённый репозиторий: `github.com/konicaRu/cover-letter-prompt`, ветка `main`, публичный.
