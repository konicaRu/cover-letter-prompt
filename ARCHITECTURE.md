# ARCHITECTURE — устройство проекта

> Структура и все изменения. Обновляется по команде `git save`.
> Правила работы — в CLAUDE.md, его не дублируем, тут только карта и changelog.

## Назначение
Генератор сопроводительных писем под вакансии (аналитик данных Д. Кочетов)
с итеративной доработкой промпта по замечаниям пользователя.

## Слои промпта (приоритет: core > corrections > rules)
- `prompt/core.md` — заблокированное ядро. Меняется только командой «изменить core».
- `prompt/rules.md` — банк формулировок (термины + обороты), справочник стиля. Пометки
  `[факт]` (шаблон с цифрой — только под факт из digest) и `[фон]` (книжный термин, не дословно).
- `prompt/corrections.md` — правки пользователя (append-only, применяются только `active`).
- `prompt/patterns.md` — справочник аналитических паттернов (Парето, медиана vs среднее,
  корреляция≠причинность, метрики тщеславия, YoY, когорты, A/B-тест и т.д.): вставлять
  ТОЛЬКО по реальному якорю, макс. 1 на письмо. Активируется C-048. Исключён из git (якоря А3).
- `prompt/humanizer.md` — флаг очеловечивания (одна строка: `on`/`off`, по умолч. `on`).

## Вход / выход
- `inputs/resume/` — резюме (`.md`), источник фактов №1. PDF не читаем (медленно/дорого).
- `inputs/jira/` — задачи Jira, источник опыта №2.
- `inputs/facts-digest.md` — выверенный дистиллят кейсов из резюме+Jira с цифрами и привязкой
  к источнику; источник ПРОВЕРКИ фактов (C-040). Исключён из git.
- `inputs/cases.md` — банк готовых кейсов для ПРИМЕРОВ в письмах (пул на базе Jira,
  Jira-приоритет; резерв из резюме — только если Jira не закрывает домен). Активируется
  C-041. Исключён из git (данные А3).
- `inputs/vacancy.md` — текущая вакансия.
- `outputs/letters.md` — варианты письма (плоская версия), по умолч. 2.
- `outputs/letters_humanized.md` — те же письма после скилла `humanizer` (когда флаг `on`).
- `outputs/letters_edit.md` — площадка «режима редакции»: пользователь правит письма прямо
  в редакторе, пишет `ок` — правки разбираются в правила.
- `bench/` — эталонные ПАРЫ вакансия+письмо для регресс-проверки. Одна папка на вакансию
  по короткому имени работодателя (`bench/gri/` = vacancy.md + letters.md).

## Служебные файлы
- `MEMORY.md` — журнал диалога: на чём закончили.
- `ARCHITECTURE.md` — этот файл.
- `archive/applications.md` — архив пар «вакансия → отправленное письмо» для статистики
  (append, свежие сверху; полный текст вакансии + письмо + тег-заголовок). Папка `archive/`
  исключена из git. Заполняется при выборе письма (CLAUDE.md → «Архив откликов»).

## Команды
- «Сгенерируй письмо» (по умолч. 2 варианта + строка времени генерации), «Замечание: …»,
  «Правка письма / режим редакции» (правка в letters_edit.md → `ок` → правила),
  «Проверь на противоречия», «Повысить C-NNN до core», «humanizer on/off» — описаны в CLAUDE.md.
- `git save` — флаш + бэкап: (1) обновить MEMORY.md и ARCHITECTURE.md актуальным
  состоянием; (2) `git add -A` (inputs/jira исключён через .gitignore), commit, `git push origin main`.
  Репо: github.com/konicaRu/cover-letter-prompt (main, ПУБЛИЧНЫЙ). Перед коммитом —
  sanity-check, что Jira не в staged.
- `git load` — утреннее восстановление: прочитать MEMORY.md + active-правки + humanizer
  и кратко доложить, где остановились. Только чтение, ничего не пишет.
