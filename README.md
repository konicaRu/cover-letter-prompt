# Cover Letter Prompt — генератор сопроводительных писем

Проект для генерации сопроводительных писем под вакансии аналитика данных
(кандидат — Дмитрий Кочетов) с **итеративной доработкой промпта** по замечаниям.
Каждое замечание превращается в атомарное правило, правила копятся и улучшают
качество писем. Параллельно собран «экспортный» промпт для сторонних нейросетей
(ChatGPT) — см. [`external-prompts/`](external-prompts/).

> Рабочий процесс ведёт ИИ-ассистент (Claude Code) по инструкции в
> [`CLAUDE.md`](CLAUDE.md). Этот README — обзор; детали правил — там.

## Идея
Письмо не пересказывает резюме, а отвечает на вопрос работодателя «почему именно
этот аналитик решит наши задачи». Принцип: **доказывать делом, а не заявлять** —
формула «проблема → что сделал → метрика → польза бизнесу». Факты берутся только
из резюме и задач Jira; ничего не выдумывается.

## Как это работает: слои промпта
Итоговый промпт собирается из трёх слоёв (приоритет при конфликте: **core > corrections > rules**):

| Файл | Роль |
|---|---|
| [`prompt/core.md`](prompt/core.md) | Заблокированное ядро. Меняется только командой «повысить C-NNN до core». |
| [`prompt/rules.md`](prompt/rules.md) | Правила стиля и промптинга (справочник). |
| [`prompt/corrections.md`](prompt/corrections.md) | Правки пользователя C-001…C-014 (append-only, применяются только `active`). |
| [`prompt/humanizer.md`](prompt/humanizer.md) | Флаг «очеловечивания» (`on`/`off`, по умолч. `on`). |

## Структура репозитория
```
cover-letter-prompt/
├─ README.md              ← этот файл
├─ CLAUDE.md              рабочая инструкция (правила проекта)
├─ MEMORY.md              журнал: где остановились и почему
├─ ARCHITECTURE.md        карта проекта + changelog
├─ .gitignore             исключает inputs/jira из git
├─ prompt/                слои промпта (core, rules, corrections, humanizer)
├─ inputs/
│  ├─ resume/             резюме (.md/.pdf) — источник фактов №1
│  ├─ jira/               задачи Jira — источник №2 (локально; в git НЕ публикуется)
│  └─ vacancy.md          текущая вакансия
├─ outputs/
│  ├─ letters.md          3 варианта письма (плоские)
│  └─ letters_humanized.md  3 варианта (humanized)
├─ bench/gri/             эталон для регресс-проверки (пара вакансия+письмо)
└─ external-prompts/      промпт для сторонних нейросетей + методология
```

## Вход / выход
- **Резюме** — `inputs/resume/` (основной источник фактов).
- **Jira** — `inputs/jira/` (полноценный источник опыта; в git не уходит).
- **Вакансия** — `inputs/vacancy.md` (текущая).
- **Письмо** — 3 варианта в чат и в `outputs/letters.md` (+ humanized-версия).

## Команды
| Команда | Что делает |
|---|---|
| «Сгенерируй письмо» | Собрать промпт, разобрать вакансию, выдать ровно 3 варианта. |
| «Замечание: …» | Превратить замечание в атомарное правило, проверить на противоречия, добавить, перегенерировать. |
| «Проверь на противоречия» | Прогнать `active`-правки между собой и против ядра. |
| «Повысить C-NNN до core» | Перенести правку в ядро (единственный способ изменить core). |
| «humanizer on / off» | Переключить живой стиль (по умолчанию on). |
| `git save` | Обновить MEMORY/ARCHITECTURE + закоммитить и запушить бэкап на GitHub. |
| `git load` | Утром восстановить контекст из MEMORY + правки + флаг. |

## Накопленные правки (C-001…C-014)
Полные формулировки — в [`prompt/corrections.md`](prompt/corrections.md). Кратко:
- **C-001** короткие предложения и переносы строк внутри абзацев.
- **C-002** прошедшее время для опыта.
- **C-003** живые фразы с подводкой и пользой, без повисших номинативов.
- **C-004** понятно постороннему; инструмент + что им делал, простым языком.
- **C-005** не склеивать БД и язык запросов («из ClickHouse на SQL» — нельзя).
- **C-006** полный блок контактов в КАЖДОМ письме.
- **C-007** для агентств имя компании 2× не требуется; anchor по домену.
- **C-008** только подтверждённые факты; не раздувать учебное/прогнозное; не переименовывать опыт.
- **C-009** никогда не упоминать ГНБ/стройку и маркетинг/GA-Метрику оттуда.
- **C-010** если требуется высшее — иронично признать отсутствие + перевести на курсы/опыт.
- **C-011** современный разговорный стиль, без канцелярита.
- **C-012** от первого лица («Моя сильная сторона…»).
- **C-013** связность письма, без оборванных/двусмысленных фраз.
- **C-014** в каждом письме подчёркивать самостоятельность (senior vs middle).

## Регресс-бенч
В [`bench/`](bench/) лежат эталонные **пары вакансия+письмо**. После новой правки,
если бенч не пуст, текущий промпт прогоняется по эталонам — проверка, что удачные
письма не стали хуже. Сейчас эталон один: `bench/gri/`.

## Промпт для сторонних сервисов
[`external-prompts/`](external-prompts/) — самодостаточный промпт для ChatGPT
(факты берёт из загруженных резюме/Jira), правила его сборки (`building-rules.md`)
и конспект гайда OpenAI по промптингу GPT-5.

## Бэкап и конфиденциальность
- Репозиторий: `github.com/konicaRu/cover-letter-prompt` (ветка `main`, публичный).
- `.gitignore` исключает `inputs/jira/` — это внутренние данные бывшего
  работодателя, они **не публикуются**. Перед каждым коммитом — проверка, что
  Jira не попала в staged.

## Принципы работы (по А. Карпатому)
Думать до действия (при неоднозначности — спросить, противоречия выводить),
минимализм, точечные изменения, работа от критерия готовности. Подробнее — в
[`CLAUDE.md`](CLAUDE.md).
