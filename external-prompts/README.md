# external-prompts

Промпты и методология для сторонних нейросетей (ChatGPT, Claude и т.п.), которые
должны генерировать сопроводительные письма вне этого проекта.

## Что внутри
- [`agent-handoff.md`](agent-handoff.md) — **памятка переноса (read me first).** Один файл,
  который даёшь новому агенту со словами «прочти это»: что за проект, что читать, какие команды,
  что не переносится. Точка входа.
- [`cover-letter-chatgpt.md`](cover-letter-chatgpt.md) — готовый системный промпт
  (образец). Вставляется в Instructions проекта ChatGPT / в `AGENTS.md` для Codex;
  факты берутся из загруженных файлов резюме и Jira. В чат подаётся текст вакансии →
  на выходе 2 варианта письма. Актуален на 2026-06-09 (ядро: 3 сегмента, релевантность кейсов).
- [`building-rules.md`](building-rules.md) — правила, по которым собран этот промпт:
  выжимка из правок C-001…C-032 и принципов проекта (раздел 10 — обновления и
  изменения ядра). Используется, чтобы пересобрать/доработать промпт под другого
  кандидата или сервис.
- [`gpt5-prompting-guide.md`](gpt5-prompting-guide.md) — конспект официального гайда
  OpenAI по промптингу GPT-5 (структура промпта, контроль «усердия», self-check
  и т.д.), по которому оформлен сам промпт.

## Перенос на Codex / другую нейросеть
Переносится только КОНТЕНТ (это обычный Markdown). Харнес-фишки Claude Code НЕ
переносятся: команды «git save / git load / Замечание / режим редакции», скилл
`humanizer`, звуковой хук, MCP, авто-память. Что сделать:
1. `cover-letter-chatgpt.md` → в системные инструкции модели (ChatGPT — Instructions
   проекта; Codex — `AGENTS.md` в корне репо, т.к. `CLAUDE.md` он сам не читает).
2. Загрузить рядом факты: резюме и (по возможности) Jira; либо вшить выверенный
   дистиллят. Без фактов промпт писать не должен.
3. Humanizer уже вшит в промпт (блоки `<живость_и_голос>` и `<стиль>`) — отдельный
   скилл не нужен.
4. При добавлении новых правок C-NNN в проекте — синхронизировать сюда вручную
   (этот дистиллят не обновляется автоматически).

## Источник методологии
Основной движок и история правок — в корне проекта (`prompt/`, `CLAUDE.md`,
`prompt/corrections.md`). Эта папка — «экспортный» дистиллят для внешних сервисов.
